Apache SkyWalking .NET core 探针发布！

  

GitHub: https://github.com/apache/incubator-skywalking  
码云Gitee: https://gitee.com/OpenSkywalking/sky-walking

Apache SkyWalking在4月初，发布了加入Apache孵化器后的第一个版本：5.0.0-alpha。虽然只是一个alpha预览版本，依然得到的社区的强烈回应，多加公司快速跟进，升级或者新部署的新版本。现代化UI和丰富全面的分析能力得到社区的肯定。

截止目前，已经有多加公司主动向社区分享了他们的案例。我们也期待更多的公司，分享部署案例，支持项目的发展。点击这里提交：https://github.com/apache/incubator-skywalking/issues/443

随着SkyWalking生态的发展，在今年年初开始于.NET CORE COMMUNITY的合作，经过4个月的孵化，在今天发布SkyWalking生态的.NET Core探针的0.1.0版本。

项目地址：https://github.com/OpenSkywalking/skywalking-netcore

.NET Core探针遵守SkyWalking追踪协议，能够对.NET Core生态的应用进行监控，并结合SkyWalking的分析追踪能力提供数据聚合，持久化，告警和可视化等能力。当然，由于.NET Core探针的首次发布，需要下载我们的后端定制版（配置定制）来配合使用。SkyWalking社区将在5.0.0-beta发布时，完全包含对.NET Core探针的支持，确保native support。

.NET Core探针的路线图中，将通过两到三个版本，增加.NET Core自动探针程度，能够无感知增强。并进一步加强和后端的融合，增加除Trace数据之外的性能指标监控。

同时，官方透漏，NodeJS服务端自动探针，也会在下个月发布。GO手动探针也在建设中（也希望有GO社区的同学来积极帮忙）。到那时，SkyWalking将开始覆盖多语言场景，提供多语言的前面监控。
